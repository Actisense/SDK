# NMEA 0183 Data Format

Note: The NMEA 0183 standard is both a protocol and a data format. See the [NMEA 0183 protocol document](../../DataProtocols/nmea0183_protocol.md) for details of how NMEA 0183 frames are decoded into sentences.

## Description

Once decoded, NMEA 0183 frames are called `Sentences`

## Sentence Structure

The main components of this sentence are:

- **Start Character**: `$` or `!`
- **Talker Identifier**: Two characters that identify the device sending the data.
- **Sentence Identifier**: Three characters that define the type of data being sent.
- **Data Fields**: Comma-separated values that contain the actual data.

### Example sentence breakdown:

The NMEA 0183 frame arriving in the serial stream is:

`$GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,*47<CR><LF>`

Once the protocol overhead (checksum `*47` and delimiters `<CR><LF>`) is removed, the remaining sentence data is:

`$GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,`

#### Field-by field description:

* `$` Start character
* `GP` Talker ID - GPS
* `GGA` Sentence formatter
* `123519` Time (12:35:19 UTC)
* `4807.038,N` Latitude (48°07.038' N)
* `01131.000,E` Longitude (11°31.000' E)
* `1` Fix quality (1 = GPS fix)
* `08` Number of satellites being tracked
* `0.9` Horizontal dilution of position
* `545.4,M` Altitude (545.4 meters above mean sea level)
* `46.9,M` Height of geoid above WGS84 ellipsoid
